## Common Scenarios

### Scenario 1: Adding a New Local-Only Directory
```bash
# Add to .gitignore
echo "my-local-tool/" >> .gitignore

# Verify it's ignored
git check-ignore -v my-local-tool/
```

### Scenario 2: Accidentally Committed Node Modules
```bash
# Remove from git without deleting files
git rm --cached -r node_modules/

# Verify
git status  # Should show node_modules/ as untracked (ignored)
```

### Scenario 3: New Environment Variable Needed
```bash
# Add to .env (never committed)

# Update .env.example (committed as template)
```

## Best Practices

### Do's ✅
- Always add sensitive files to `.gitignore`
- Use `.env.example` as template, never commit `.env`
- Add generated files to ignore
- Add node_modules and dependencies
- Review `.gitignore` before major changes

### Don'ts ❌
- Never commit `.env` files with secrets
- Don't ignore source code files
- Don't ignore `package.json` or essential configs
- Don't commit generated reports to main branch
- Don't override `.gitignore` without team discussion
